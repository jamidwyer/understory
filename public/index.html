Understory

A catalog of every component in @jamidwyer/simple-components — live examples and the HTML that produced them.

<uwc-button>

Button with three variants, or a disabled state.

Variants

Primary Secondary Danger Disabled 
<uwc-button variant="primary">Primary</uwc-button>
<uwc-button variant="secondary">Secondary</uwc-button>
<uwc-button variant="danger">Danger</uwc-button>
<uwc-button variant="primary" disabled>Disabled</uwc-button>

<uwc-link-button>

Styled identically to <uwc-button>, but renders a real <a href> for navigation.

Variants

Primary link Secondary link Disabled link 
<uwc-link-button variant="primary" href="https://example.com" target="_blank" rel="noopener">
  Primary link
</uwc-link-button>
<uwc-link-button variant="primary" disabled>Disabled link</uwc-link-button>

<uwc-input>

Text input with built-in HTML5 validation. Built to be subclassed — see Password Input and Phone Input below.

Validated fields

<uwc-input label="Email" type="email" placeholder="you@example.com" required
  error-message="Enter a valid email"></uwc-input>

Validation inferred from autocomplete

<!-- no "type" set: inferred from autocomplete -->
<uwc-input label="Card number" autocomplete="cc-number" placeholder="4242 4242 4242 4242" required></uwc-input>

<uwc-password-input>

Extends <uwc-input>: always type="password", with an eyeball button inside the field to show/hide the value.

Standalone example

<uwc-password-input label="Password" placeholder="At least 8 characters"
  required minlength="8"></uwc-password-input>

<uwc-phone-input>

Extends <uwc-input>: defaults to type="tel" and autocomplete="tel", bringing in the same phone-number pattern and numeric keyboard hint as <uwc-input autocomplete="tel">.

Standalone example

<uwc-phone-input label="Phone" placeholder="(555) 123-4567" required></uwc-phone-input>

<uwc-image>

Loads any source image into a fixed width/height box without distorting it — cropped (cover) or letterboxed (contain).

fit="cover" vs fit="contain"

<uwc-image src="/landscape.jpg" alt="Wide landscape" width="160" height="160"></uwc-image>
<uwc-image src="/landscape.jpg" alt="Wide landscape" width="160" height="160" fit="contain"></uwc-image>

<uwc-card>

Content card with an optional full-bleed image, a fixed height, and heading/subtitle text that clamps with an ellipsis rather than overflowing.

With image, and overflowing text

<uwc-card
  heading="Lorem ipsum dolor sit amet consectetur adipiscing elit..."
  subtitle="Duis aute irure dolor in reprehenderit in voluptate velit esse..."
  image="/photo.jpg"
></uwc-card>

<uwc-card-container>

Lays out cards in a flex row that wraps: cards grow to fill the available width, wrapping once a card would shrink past its own min-width.

Three cards, wrapping

<uwc-card-container gap="1rem">
  <uwc-card heading="Lorem ipsum dolor" subtitle="Sit amet..." image="/1.jpg" min-width="12rem"></uwc-card>
  <uwc-card heading="Sed do eiusmod" subtitle="Tempor incididunt..." image="/2.jpg" min-width="12rem"></uwc-card>
  <uwc-card heading="Ut enim ad minim" subtitle="Veniam quis..." image="/3.jpg" min-width="12rem"></uwc-card>
</uwc-card-container>

<uwc-dropdown-menu>

Button that toggles a panel of links, each with an optional description. Closes on outside click, Escape, or after picking a link.

Resources menu

<uwc-dropdown-menu
  label="Resources"
  items='[
    {"href":"/docs","label":"Docs","description":"Guides and API reference"},
    {"href":"/support","label":"Support","description":"Get help from the team"}
  ]'
></uwc-dropdown-menu>

<uwc-sidebar-menu>

Vertical navigation menu that highlights the currently selected link. This page's own left sidebar is one.

Standalone example

<uwc-sidebar-menu
  links='[{"href":"/overview","label":"Overview"},{"href":"/start","label":"Getting started"}]'
  selected="/start"
></uwc-sidebar-menu>

<uwc-universal-nav>

Full-width top bar with 20px padding: a flex row with branding and user/login named slots, justified space-between and centered. This page's own header is one. The logged-in attribute switches between the two: logged in shows the user slot, logged out shows the login slot (falling back to a default "Log in" button if none is given).

Logged in

Acme Sign out 
<uwc-universal-nav logged-in>
  <strong slot="branding">Acme</strong>
  <uwc-button slot="user" variant="secondary">Sign out</uwc-button>
</uwc-universal-nav>

Logged out

Acme 
<uwc-universal-nav>
  <strong slot="branding">Acme</strong>
  <!-- no user/login content given: falls back to a default "Log in" button -->
</uwc-universal-nav>
